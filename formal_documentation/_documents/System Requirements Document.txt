System Requirements Document (SRD): Project Vector_ThreatID

1. Functional Requirements (FR)
	FR-1: Automated Framework Ingestion: The system must ingest MITRE ATT&CK, CVE, and OWASP data from local JSON/CSV files and convert them into high-dimensional vectors.
	FR-2: Semantic Log Analysis: The system must process raw logs (Syslog/JSON) and map them to the nearest "Threat Anchor" using cosine similarity.
	FR-3: Dynamic Risk Scoring: A real-time risk score ($0.0$ to $1.0$) must be generated for every log line based on vector distance and metadata weights (e.g., CVSS score).
	FR-4: Contextual RAG Retrieval: Upon hitting a risk threshold ($> 0.85$), the system must retrieve a specific SOP from the local knowledge base.
	FR-5: Metadata-Driven Filtering: SOP retrieval must be filtered by Environment, Access Level, and Assigned Team to ensure accuracy.

2. Non-Functional Requirements (NFR)
	NFR-1: Air-Gapped Operation: All embedding and vector search operations must run locally without external API calls.
	NFR-2: Scalability: The system must handle a burst rate of $10,000$ logs per second using a decoupled message broker (Kafka).
	NFR-3: Latency: End-to-end processing (Ingestion to Alert) must occur in $< 500$ milliseconds.

	NFR-4: Auditability: Every alert must include a "Mathematical Trace"—the exact distance and the top 3 framework matches that triggered the alert.
